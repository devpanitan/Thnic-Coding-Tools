เสาที่ 1 · Decomposition

แตกปัญหาใหญ่ ให้เป็นงานเล็ก ๆ

ปัญหา “ทายเลขลับให้ไวที่สุด” ดูยาก แต่ถ้าแตกเป็นงานย่อย จะทำทีละก้าวได้ กดปุ่มเพื่อค่อย ๆ แตกปัญหาออกมา

เฉลี่ยราว ๆ 50 ครั้ง! เพราะเดามั่วตัดตัวเลือกทิ้งได้ทีละ 1 ตัว → เดี๋ยวเสาที่ 4 เราจะทำให้เหลือแค่ ~7 ครั้ง

เสาที่ 2 · Pattern Recognition

มองหา “สิ่งที่ทำซ้ำ” เดิม ๆ

ถ้าดูงานย่อย B→C→D จะเห็นว่าทุกก้าวเราทำ 3 อย่างเดิมซ้ำ ๆ พอเจอแบบซ้ำ เราก็เขียนเป็นวงวน (loop) ได้เลย

สรุปเสา 2: เจอแบบซ้ำ = โอกาสเขียนเป็น loop — ไม่ต้องเขียนคำสั่งเดิมซ้ำ ๆ หลายรอบ
เสาที่ 3 · Abstraction

คัดเฉพาะสิ่งสำคัญ ตัดที่ไม่จำเป็นทิ้ง

เกมจริงมีรายละเอียดเยอะ (สี เสียง ชื่อคนเล่น) แต่ในการแก้ปัญหา เราสนใจแค่ไม่กี่อย่าง กดการ์ดเพื่อจัดว่า “เก็บ” หรือ “ตัดทิ้ง” แล้วกดตรวจ

แตะ 1 ครั้ง = ✓ เก็บ · แตะซ้ำ = ✕ ตัดทิ้ง · แตะอีกครั้ง = ล้าง

สรุปเสา 3: เหลือแค่ 3 อย่างที่จำเป็นจริง ๆ คือ lo, hi, mid → กลายเป็น “ตัวแปร” ในโปรแกรมพอดี
เสาที่ 4 · Algorithm

เรียงเป็นลำดับขั้นตอน → กลายเป็นโปรแกรม

เอา 3 เสาก่อนหน้ามาร้อยเป็นขั้นตอนชัดเจน นี่แหละคือ Binary Search ลองกด “เดาเลขกลาง” ดูว่ากี่ครั้งถึงเจอ (เลขลับสุ่มไว้แล้ว 1–100)

1255075100
เดาไปแล้ว 0 ครั้ง
ช่วงตอนนี้: 1 – 100 (มี 100 ตัวเลขที่เป็นไปได้)
#include <iostream> using namespace std; int main() { int secret = 73; // เลขลับที่ต้องหา (1..100) int lo = 1, hi = 100; // เสา 3: เก็บแค่ lo, hi int rounds = 0; // นับจำนวนครั้งที่เดา while (lo <= hi) { // เสา 2: ทำซ้ำ = loop int mid = (lo + hi) / 2; // หาเลขกลาง rounds++; if (mid == secret) { // เทียบ: เจอแล้ว cout << "เจอเลข " << mid << " ใน " << rounds << " ครั้ง\n"; break; } else if (mid < secret) { // ต่ำไป → ตัดครึ่งล่างทิ้ง cout << mid << " ต่ำไป\n"; lo = mid + 1; } else { // สูงไป → ตัดครึ่งบนทิ้ง cout << mid << " สูงไป\n"; hi = mid - 1; } } return 0; }

💡 โค้ดนี้รันได้จริง — คัดลอกไปลองที่ OneCompiler แล้วลองเปลี่ยนค่า secret ดู

สรุป

4 เสา ทำงานร่วมกันเป็น “โปรแกรม”

จากปัญหาที่ดูยาก → แตกย่อย → หาแบบซ้ำ → คัดสิ่งสำคัญ → เรียงเป็นขั้นตอน = เขียนโค้ดได้จริง

แตกย่อยปัญหาใหญ่ → งานเล็ก
หาแบบซ้ำงานซ้ำ → loop
คัดสำคัญเหลือ lo, hi, mid
ลำดับขั้น= Binary Search

ลองนำไปเล่นต่อกับเครื่องมืออื่น ๆ ของเรา 👇
