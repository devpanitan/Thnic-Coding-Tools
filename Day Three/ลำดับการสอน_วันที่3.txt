------------------------------------------------------------
ช่วงเช้า — การวนซ้ำ (Loop)
------------------------------------------------------------
1) [สไลด์ 1–2] เปิดคาบ + ทบทวนเมื่อวาน (ตัวแปร/operators/if-else)
   - ถามเด็ก: เมื่อวานเขียนอะไรได้บ้าง

2) [สไลด์ 3] ตั้งปัญหา "อยากพิมพ์ 1 ถึง 1000 จะทำยังไง"
   - ให้เห็นว่าเขียน cout พันบรรทัดไม่ไหว → ต้องมีการวนซ้ำ

3) [สไลด์ 4–5] สอน for loop + แยก 3 ส่วน (เริ่ม/เงื่อนไข/เพิ่ม)
   → เปิด simulator_loop.html เดินทีละรอบให้เด็กดู i เปลี่ยนค่า
   - กดหยุดทีละสเต็ป ถามเด็กว่า "รอบต่อไป i เป็นเท่าไร พิมพ์อะไร"

4) [สไลด์ 6–7] while loop + เปรียบเทียบ for vs while
   - ย้ำ: ลืมเพิ่มค่า = วนไม่รู้จบ (infinite loop)

5) [สไลด์ 8] ตัวอย่างใช้จริง: บวก 1..100 = 5050
   → ใช้ simulator_loop.html โหมดผลรวม ให้เห็น sum สะสม

6) *กิจกรรม* เกม_ทำนายผลลูป.html
   - แบ่ง 2 ทีม (หรือมากกว่า) ผลัดกันทายผลลูป ตอบถูกได้คะแนน
   - คุมเวลา ~10–15 นาที สนุก ๆ ก่อนพัก

------------------------------------------------------------
ช่วงบ่าย — อาเรย์ (Array)
------------------------------------------------------------
7) [สไลด์ 9] อาเรย์คืออะไร — กล่องเรียงต่อกัน index เริ่ม 0
   → เปิด simulator_array.html ใส่ค่าในกล่อง ให้เห็น index กำกับ

8) [สไลด์ 10] เข้าถึงสมาชิก score[i]
   → ใน simulator_array.html กดปุ่ม "เข้าถึง" ลองหลาย index
   - ลองใส่ index เกินขอบเขต ให้เห็นคำเตือน (out of range)

9) [สไลด์ 11] Loop + Array — วนอ่านทั้งชุด
   → ใน simulator_array.html กด "วน for ทั้งหมด" ดูไล่ทีละช่อง

10) [สไลด์ 12–13] สรุปเครื่องมือ + สรุปเนื้อหา + โยงพรุ่งนี้ (โอลิมปิก)

11) *วัดผล* ใบงานดิจิทัล_วันที่3.html
    - เด็กกรอกชื่อทีม ทำ 10 ข้อ (เติมคำ/ทำนายผล/หา error)
    - ตอบครบ → กดตรวจครั้งเดียว → คะแนนเข้า Google Sheet อัตโนมัติ
    - ★ โจทย์ท้าย: เขียน for พิมพ์เลขคู่ 2–10 ใน OneCompiler

------------------------------------------------------------
เคล็ดลับการสอน
------------------------------------------------------------
- ใช้ simulator "หยุด-อธิบาย" บ่อย ๆ ให้เด็กทายก่อนเฉลย
- ย้ำจุดพลาดบ่อย: index เริ่ม 0, i < n (ไม่ใช่ i <= n), ลืมเพิ่มค่า
- เด็กเร็ว → ให้ลองโจทย์ท้ายในเว็บ, เด็กช้า → จับคู่ช่วยกัน
- ดูคะแนนรายข้อในชีต เพื่อรู้ว่าข้อไหนพลาดเยอะ ค่อยทวนเพิ่ม
